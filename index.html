The Story

三百块钱的念头

01

我不想交那三百块

2024 年底，互站网上架一套电商源码，审核过了，弹出来：请缴纳 300 元保证金。

我没交。不是交不起。三百块也就是一顿好点的饭钱。但我就觉得这事特别拧巴——我卖代码，代码都还没人买呢，先让我掏钱？凭什么。

然后我翻了一下互站网上的商城源码，前二十名全是 PHP。不是 PHP 不好，但那些代码吧……没有类型检查、没有测试、SQL 拼接、admin 密码写死在配置文件里。2024 年了，还在用十年前的写法卖钱。

我当时就想：那我写一套吧。用现在该用的技术栈，把测试写好，把安全做扎实。不为别的，就是想证明这事能做得更好。

02

写代码容易，写对难

前两周特别爽。搭项目、选技术栈、写数据模型，每天写到凌晨一两点，不觉得累。两周跑通了核心流程——注册、商品、购物车、下单。

然后我就飘了。心想「这不挺简单的吗」。

接着购物车的并发 bug 就来了。两个请求同时改同一件商品数量，数据直接乱掉。修了一版，觉得好了。一跑测试，又一个边界情况。再修，再崩。改了快两周，几乎没推进任何新功能。

那段时间每天深夜看着满屏报错，心里反复在想：我一个人真的能把这个东西做完吗？

03

安全感是自己给的

代码写多了，恐惧是反的——不是怕功能做不出来，是怕做出来的东西被别人部署到公网上出安全事故。

我开始一层一层加防护：JWT 双 token、六层 Rate Limit、CSRF 防护、文件上传 magic bytes 校验……每次加一层，之前的代码就得重构一遍。最崩溃的一次改了认证中间件，五十七个 API 端点全部要改调用方式。那天从下午改到凌晨四点，中间吃了一碗泡面。

有人问我：你一个人写这么多安全措施干嘛？

我说：因为我知道我的代码会被人部署到公网上。我不能让它出事。

04

能跑了，就交付吗？

写到第三十几轮的时候，功能其实早就全了。换个心态的人，可能就上线开卖了。

但我没忍住。每次觉得「差不多了」，就又发现一个想改的地方：这个变量名不够好、那个函数的错误处理太糙、这块逻辑能再简化三层嵌套……六十轮不是计划出来的，是我每次觉得"差不多了"然后又改了一轮，改着改着就到六十了。

最后那天跑了一次完整测试——250 个用例，32 秒全绿。那个瞬间比任何夸奖都让人安心。因为我知道，每一行关键代码背后都有一个盯着它的测试。

"我不是公司，不是团队，只是一个觉得市面上的电商源码不够好的开发者。六十轮迭代之后，我觉得它够好了。"
— 栖物的创建者，2025 年 5 月

FAQ

你可能想问的

不懂 Node.js 能部署吗？

能。Docker 一条命令部署，也支持传统 npm 方式。选购安装服务可远程协助完成。

源码有加密或后门吗？

完全没有。纯净 TypeScript 源码，无授权验证、无后门、无加密，100% 自由查看修改。

能用 MySQL 替代 SQLite 吗？

可以。内置 DB 抽象层，支持切换到 MySQL 或 PostgreSQL。

和互站网其他商城有什么不同？

互站网 80%+ 是 PHP 商城。栖物是极少数 Node.js + React 19 + TypeScript 全栈方案，架构领先一代。

支付怎么用？

系统预置模拟支付方便测试。对接支付宝约 30 分钟完成。

都支持。中英双语 200+ 条目，PWA 可添加到桌面离线使用。
