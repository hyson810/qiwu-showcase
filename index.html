The Story

为什么会有栖物

01

三百块钱的念头

2024 年底，我准备在互站网上架一套电商源码。审核通过后，系统提示：请缴纳 300 元保证金。我没有交。不是因为交不起，是觉得这不对——我想卖代码，为什么要先交一笔钱才能开始？那之后我开始想：有没有一个地方，源码就放在 GitHub 上，不满意直接走人，不需要押金，不需要信任成本。

于是有了栖物。

02

满屏的 PHP

互站网 80% 以上的商城源码是 PHP——ThinkPHP、Laravel，十年前的技术栈。没有一套 Node.js + React 19 + TypeScript 的全栈方案。这让我困惑：前端已经迭代到 React 19 + Vite 8 了，为什么电商源码还停在 PHP？我决定自己做。用这个时代该用的技术，写一套干净、现代、有测试、有文档的电商系统。

03

六十个深夜

从第一行代码到 v2.2.0，60 轮完整迭代。安全修了 12 轮——游客滥用优惠券、CSP 强化、JWT 续期、文件上传 magic bytes 验证。性能优化 12 轮——缓存雪崩保护、LRU 优化、前端懒加载。最累的是测试轮——我一个人写了 250+ 测试用例，从后端 CRUD 到 E2E 全链路。因为我卖的是源码，源码就该敢把测试亮出来。

04

11 层防护

卖源码最怕什么？怕被人说「不安全」。所以栖物有 11 层安全防护：JWT 32 字符强校验、6 层 Rate Limit、CSRF double-submit cookie、XSS sanitize 白名单、文件 magic bytes + MIME + 扩展名三重验证、4 级角色权限、bcrypt 12 轮哈希、CSP 生产强化、Helmet 安全头。

你可以让安全专家审计——我欢迎。

FAQ

你可能想问的

不懂 Node.js 能部署吗？

能。Docker 一条命令部署，也支持传统 npm 方式。选购安装服务可远程协助完成。

源码有加密或后门吗？

完全没有。纯净 TypeScript 源码，无授权验证、无后门、无加密，100% 自由查看修改。

能用 MySQL 替代 SQLite 吗？

可以。内置 DB 抽象层，支持切换到 MySQL 或 PostgreSQL。

和互站网其他商城有什么不同？

互站网 80%+ 是 PHP 商城。栖物是极少数 Node.js + React 19 + TypeScript 全栈方案，架构领先一代。
